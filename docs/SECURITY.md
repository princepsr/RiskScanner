# Security Policy

## Reporting a vulnerability

If you believe you have found a security vulnerability in BuildAegis, please report it responsibly.

- Do not open a public GitHub issue for security reports.
- Prefer GitHub Security Advisories (private disclosure) for the repository.

If you cannot use GitHub Security Advisories, you may contact the maintainer via the GitHub profile associated with this repository.

## Supported versions

Only the latest `main` branch is supported.

## Disclosure process

- You report the issue privately.
- Maintainers validate and triage.
- A fix is prepared and released.
- Public disclosure happens after a fix is available.
